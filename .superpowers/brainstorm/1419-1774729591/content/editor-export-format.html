Export JSON Format

Editor Save Format (vollständig, für Re-Import)
{
  "name": "Classic",
  "canvasSize": [1200, 700],

  "vertices": {
    "v0": [120.0, 30.5],
    "v1": [180.0, 45.0],
    "v2": [200.0, 80.0]
  },

  "edges": {
    "e0": ["v0", "v1"],
    "e1": ["v1", "v2", "v5", "v8"],
    // ^ Polylinie: v1 → v2 → v5 → v8
  },

  "territories": {
    "Alaska": {
      "edges": ["e0", "-e3", "e7"],
      // ^ "-" = Kante rückwärts traversieren
      "color": "#E53935",
      "labelPosition": [60, 85]
    }
  },

  "continents": {
    "North America": {
      "bonus": 5,
      "territories": ["Alaska", "NW Territory"]
    }
  },

  "manualAdjacencies": [
    ["Alaska", "Kamchatka"]
  ],
  "manualNonAdjacencies": [
    ["Brazil", "West Africa"]
  ],

  "backgroundImage": null
  // ^ Wird NICHT exportiert, nur im Editor-Save
}
Game Export (für Flutter-App)
{
  "name": "Classic",
  "canvasSize": [1200, 700],

  "territories": {
    "Alaska": {
      // Pfad als Punkt-Liste (aufgelöst)
      "path": [
        [120, 30], [180, 45], [200, 80],
        [260, 120], [240, 180], [180, 200],
        [120, 180], [80, 120]
      ],
      "color": "#E53935",
      "labelPosition": [160, 110]
    }
  },

  "continents": [
    {
      "name": "North America",
      "bonus": 5,
      "territories": ["Alaska", "NW Territory"]
    }
  ],

  "adjacencies": [
    ["Alaska", "NW Territory"],
    ["Alaska", "Kamchatka"]
  ]
  // ^ Auto + manuell, minus non-adj
}

Zwei Export-Formate

💾 Save (Editor-Format)
Für: Weiterbearbeiten im Editor
  • Enthält Vertices, Edges, Faces (volle Topologie)
  • Hintergrundbild-Referenz (optional)
  • Alles was der Editor braucht zum Re-Import
  • Datei: mapname.risk.json
📤 Export (Game-Format)
Für: Flutter-App als Asset
  • Territories mit aufgelösten Punkt-Listen (kein Graph)
  • Adjacencies zusammengeführt (auto + manuell − non-adj)
  • Kontinente mit Bonus
  • Datei: mapname.json → mobile/assets/

Flutter-App Anpassungen

Was sich in der Flutter-App ändern muss
territory_data.dart
Rect → Path (aus Punkt-Liste). TerritoryGeometry bekommt List<Offset> points statt Rect rect
map_base_painter.dart
canvas.drawPath() statt canvas.drawRect()
map_overlay_painter.dart
Gleiche Änderung: Path statt Rect für Fills + Selection-Border
map_widget.dart
Hit-Testing: path.contains(point) statt rect.contains()
map_provider.dart
Neues JSON-Format laden, Pfad-Daten parsen
classic.json
Ersetzt durch Game-Export aus dem Editor
