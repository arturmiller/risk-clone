Risk Map Editor — Architektur-Überblick

Single-page Web-App (HTML/JS/Canvas), kein Framework

Projekt-Struktur

Verzeichnisstruktur
risk/
  editor/
    index.html         — Entry point
    style.css          — UI Styles
    js/
      app.js           — Entry, Event-Wiring
      canvas.js        — Render-Loop, Zoom/Pan
      graph.js         — Planar Graph (Vertices, Edges, Faces)
      tools.js         — Draw, Select, Edit Tools
      snap.js          — Vertex + Edge Snapping
      faces.js         — Face Detection (Cycle Finding)
      territories.js   — Territory/Continent Management
      history.js       — Undo/Redo Stack
      io.js            — Import/Export JSON
  mobile/
    ... (bestehende Flutter-App)

Datenmodell (im Speicher)

graph.js — Kern-Datenstruktur
vertices: Map<id, {x, y}>
Alle Eckpunkte, shared zwischen Kanten

edges: Map<id, {v1, v2, midpoints[]}>
Kanten als Polylinien (v1 → midpoints → v2)

faces: Map<id, {edgeRefs[], territory?}>
Automatisch erkannte geschlossene Flächen

territories: Map<name, {faceId, color, labelPos}>
Bestätigte Territorien mit Metadaten

continents: Map<name, {bonus, territories[]}>
Kontinent-Gruppierungen

adjacencies: Set<[t1, t2]>
Auto-generiert + manuelle Overrides
