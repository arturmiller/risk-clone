**messages.py:** Add `game_mode: str = "play"` field to `StartGameMessage`. Valid values: `"play"` (human + bots) and `"simulation"` (all bots).

**game_manager.py:** Modify `setup()` to accept `game_mode: str = "play"` parameter. Store as `self._game_mode`.

In `setup()`:
- If `game_mode == "simulation"`: Do NOT create HumanWebSocketAgent. ALL players (0 through num_players-1) get bot agents based on difficulty. Set `self.human_agent = None`. Player names: "Bot 1", "Bot 2", ..., "Bot N" (no "You"). Also add "hard" to the difficulty validation (currently only allows "easy" and "medium").
- If `game_mode == "play"`: Existing logic unchanged (player 0 = human).

Add `_run_simulation_loop()` method that is similar to `_run_game_loop()` but:
- No `is_human` check -- all turns are bot turns
- Every turn: send state update via `_send_sync(state_to_message(state, mg, prompt))` where prompt is `"{player_name}'s turn..."`
- Always apply `BOT_DELAY` sleep after each turn
- Same victory/game_over logic (but `is_human_winner` is always False)
- Same event emission via `_emit_turn_events`

Modify `start_game()`: if `self._game_mode == "simulation"`, run `_run_simulation_loop` in the thread instead of `_run_game_loop`.

**app.py:** In the WebSocket handler for `start_game`, extract `game_mode = data.get("game_mode", "play")` and pass it to `manager.setup(game_mode=game_mode)`. Also add "hard" to the difficulty handling in the setup flow.
