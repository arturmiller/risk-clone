    1. _border_security_ratio(state, territory) -> float: enemy_adjacent_armies / own_armies
    2. _opponent_threat_scores(state) -> dict[int, float]: rank opponents by army count + continent progress
    3. _continent_scores(state) -> dict[str, float]: fraction owned per continent with exponential boost >50%
    4. choose_reinforcement_placement: Concentrate ALL armies on 1-2 most vulnerable borders (highest BSR), preferring territories in highest-scoring continent
    5. choose_attack: Multi-priority ordering (continent-complete > block-opponent > high-value with probability check > favorable odds), stop when army reserve low or card already earned and marginal attacks remain
    6. choose_card_trade: Hold until 4 cards or escalation >= 4 (10+ armies), trade best set when trading
    7. choose_advance_armies: Advance more into exposed territories (many enemy neighbors), all-but-1 from interior sources
    8. choose_fortify: Move from interior (highest armies) to most vulnerable border (highest BSR), using connected path

    REFACTOR phase - Clean up, ensure consistent style with MediumAgent
  </behavior>
  <implementation>
    Follow the research architecture patterns exactly. Key implementation notes:
    - Use ATTACK_PROBABILITIES dict for probability-based attack decisions (estimate multi-round outcomes from single-roll probabilities)
    - For attack probability over multiple rounds, use geometric approximation: P(win_territory) approx from army ratio and per-roll probabilities
    - Continents >50% owned get exponential boost (score * 1.5) for prioritization
    - "Stop attacking" heuristic: stop if total border armies < 1.5 * adjacent enemy armies, or if already earned a card and no attack has >= 70% win probability
    - All strategy computed fresh from GameState each call (no persistent state per decision)
    - _best_trade helper: when multiple valid sets exist, prefer the set containing a territory card matching an owned territory (bonus 2 armies per match)
