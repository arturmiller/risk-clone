## Decisions Made

- **abstract class required for freezed 3.x** — The research and PLAN.md both used `class Foo with _$Foo` (non-abstract). In freezed 3.x, the generated mixin `_$Foo` declares abstract members (`toJson()`, field getters) that the user class must not implement (the generated `_Foo` subclass implements them). Using non-abstract `class` caused compiler errors. Fix: prefix all @freezed classes with `abstract`.
- **MapGraph uses Set<String> not List<String> for adjacency** — O(1) contains lookup for BFS inner loop; avoids duplicate edges from bidirectional insertion
- **mapGraphProvider uses MapGraphRef** — riverpod_generator 4.x generates `MapGraphRef` as the ref type; code-gen handles it correctly
