  <behavior>
    EasyAgent.chooseReinforcementPlacement:
    - Returns ReinforcePlacementAction where all keys are owned by currentPlayerIndex
    - Sum of placement values equals `armies` parameter
    - Distributes armies one-at-a-time using _rng (same logic as Python RandomAgent)

    EasyAgent.chooseAttack:
    - Returns null when player has no territories with armies >= 2
    - Returns null when no neighbors are enemy-owned
    - Returns AttackAction with source.owner == player, source.armies >= 2, target in neighbors(source), target.owner != player
    - numDice = min(3, source.armies - 1)
    - Returns AttackAction when best_advantage > 0, null ~15% of time when all disadvantaged

    EasyAgent.chooseFortify:
    - Returns null ~50% of the time (_rng.nextDouble() < 0.5) — but FakeRandom doesn't support nextDouble; use nextInt(2) == 0 for 50% logic
    - Returns null when no sources have armies >= 2
    - When returning FortifyAction: source.owner == player, source.armies >= 2, target in connectedTerritories(source, ownedSet), target != source, 1 <= armies <= source.armies - 1

    EasyAgent.chooseCardTrade:
    - Returns null when hand has fewer than 3 cards
    - Returns first valid TradeCardsAction (3-card set) when isValidSet returns true
    - Returns null when no valid set exists in hand

    EasyAgent.chooseAdvanceArmies:
    - Returns min_armies (conservative, advance minimum)

    runGame() in simulation.dart:
    - Calls setupGame(), initializes deck (createDeck + shuffle), initializes cards map
    - Loops executeTurn() until victory == true or maxTurns exceeded
    - Returns final GameState on victory
    - Throws StateError if maxTurns exceeded
  </behavior>
