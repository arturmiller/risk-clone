**1. GameScreen modifications** (`mobile/lib/screens/game_screen.dart`):

The GameScreen needs to know the current game mode. Since GameConfig is stored in GameNotifier._gameConfig (private), the cleanest approach is to pass `GameMode` as a constructor parameter to GameScreen:

```dart
class GameScreen extends ConsumerStatefulWidget {
  final GameMode gameMode;
  const GameScreen({super.key, this.gameMode = GameMode.vsBot});
}
```

**Conditional in _GameScreenState.build():**
- Keep existing `ref.listen(gameProvider, ...)` for game-over detection — works for both modes
- SKIP the reinforce phase init block when `widget.gameMode == GameMode.simulation` (no human player to init reinforcements for)
- Update PopScope: when simulation mode, `_handlePop` should call `ref.read(simulationProvider.notifier).stop()` instead of showing "Your progress will be saved" dialog. Show: "End this simulation and return to the home screen?" (same as SimulationControlBar stop confirmation). On confirm: call `simulationProvider.notifier.stop()` and `Navigator.pop(context)`.

Modify `_selectTerritoryAt()`: When `widget.gameMode == GameMode.simulation`, the tap should ONLY call `ref.read(uIStateProvider.notifier).selectTerritory(name, gameState, mapGraph)` for inspection — this is the same call as vsBot mode, so selectTerritory already sets `selectedTerritory` in UIState. The difference is that in simulation mode, ActionPanel doesn't exist to act on the selection, so selectTerritory just drives TerritoryInspector rendering. No code change needed in selectTerritory itself.

However, add toggle-off behavior: if the tapped territory is already the selected territory, clear selection instead:
```dart
void _selectTerritoryAt(Offset svgPoint, GameState gameState, MapGraph mapGraph) {
  // ... existing hit detection ...
  final selected = hits.length == 1 ? hits.first : _showDisambiguationDialog(...);
  if (selected == null) return;

  final currentSelection = ref.read(uIStateProvider).selectedTerritory;
  if (currentSelection == selected) {
    ref.read(uIStateProvider.notifier).clearSelection();
  } else {
    ref.read(uIStateProvider.notifier).selectTerritory(selected, gameState, mapGraph);
  }
}
```

Note: The disambiguation dialog returns void currently. Refactor slightly so it calls selectTerritory inside the dialog callback (existing pattern). The toggle-off only applies to single-hit taps.

Also: When tapping empty space (hits.isEmpty), clear selection: `ref.read(uIStateProvider.notifier).clearSelection();` — this dismisses TerritoryInspector.

**3. HomeScreen modifications** (`mobile/lib/screens/home_screen.dart`):

```dart
SetupForm(onStart: (config) async {
  await ref.read(gameProvider.notifier).setupGame(config);
  if (context.mounted) {
    if (config.gameMode == GameMode.simulation) {
      ref.read(simulationProvider.notifier).start(config);

      MaterialPageRoute(builder: (_) => GameScreen(gameMode: config.gameMode)),
    );

})
```

Add import for `simulation_provider.dart`.

**IMPORTANT:** HomeScreen calls `setupGame(config)` first, then `simulationProvider.notifier.start(config)`. The `start()` method in SimulationNotifier does NOT call setupGame — this separation of concerns was established in Plan 01. Verify when reading simulation_provider.dart that start() only sets status=running and calls _runLoop().
