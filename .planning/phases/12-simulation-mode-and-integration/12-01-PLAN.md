Create `mobile/lib/providers/simulation_provider.dart` with:

1. **SimulationSpeed enum**: `slow` (1000ms delay), `fast` (200ms delay), `instant` (0ms batch)

2. **SimulationStatus enum**: `idle`, `running`, `paused`, `complete`

3. **SimulationState class** (plain Dart, not freezed — ephemeral one-shot like GameConfig):
   ```dart
   class SimulationState {
     final SimulationStatus status;
     final SimulationSpeed speed;
     final int turnCount;
     final String? error;
     const SimulationState({
       this.status = SimulationStatus.idle,
       this.speed = SimulationSpeed.fast,
       this.turnCount = 0,
       this.error,
     });
     SimulationState copyWith({SimulationStatus? status, SimulationSpeed? speed, int? turnCount, String? error}) => ...;
   }
   ```

4. **SimulationNotifier** (Riverpod 3.x `@riverpod class SimulationNotifier extends _$SimulationNotifier`):
   - `build()` returns `SimulationState()` (idle, fast, turnCount 0)
   - `start(GameConfig config)`:
     - Sets status=running
     - Calls `ref.read(gameProvider.notifier).setupGame(config)` with all players as bots
     - Calls `_runLoop()`
   - `pause()`: Sets status=paused (loop checks status each iteration and awaits resume)
   - `resume()`: Sets status=running, calls `_runLoop()`
   - `stop()`: Sets status=idle, resets turnCount to 0. Calls `ref.read(gameProvider.notifier).clearSave()` and `ref.read(gameLogProvider.notifier).clear()`
   - `setSpeed(SimulationSpeed speed)`: Updates speed in state. If switching to instant while running, current Slow/Fast loop ends naturally and instant batch kicks in on next iteration. If switching FROM instant, not supported (instant runs to completion).
   - `_runLoop()` (private):
     - While status==running:
       - If speed==instant: call `_runInstant()` and return
       - Read current gameState from gameProvider
       - Check victory — if winner, set status=complete, log "Simulation Complete: {winner} wins!", return
       - Execute one turn via `Isolate.run()` using same pattern as GameNotifier.runBotTurn() (read mapGraph, build agents, executeTurn)
       - Guard with `if (!ref.mounted) return` after Isolate.run
       - Update gameProvider state with new GameState
       - Log turn action: "Turn {N}: {playerName} ({phase})"
       - Increment turnCount
       - `await Future.delayed(Duration(milliseconds: speed == slow ? 1000 : 200))`
       - Check if status changed to paused/idle during delay — if so, return
   - `_runInstant()`:
     - Run entire game in single `Isolate.run(() => runGame(mapGraph, agents, Random()))` call
     - Guard with `if (!ref.mounted) return`
     - Update gameProvider state with final GameState
     - Set status=complete
     - Log "Simulation Complete (Instant): {winner} wins! ({turnCount} turns)"

**Key patterns to follow:**
- Use `part 'simulation_provider.g.dart';` for code generation
- Use `ref.mounted` guard after every `Isolate.run` and every `await`
- Use `ref.read(gameProvider.notifier)` to update game state (not direct state mutation)
- GameNotifier's `_processing` flag is private — SimulationNotifier manages its own loop; it should NOT call `runBotTurn()` directly but instead replicate the Isolate.run + state update pattern (since runBotTurn has its own _processing guard that could conflict)
- All bot agents created via `_buildAgents()` top-level function from game_provider.dart — BUT since that's private, duplicate the agent construction logic or make it a shared utility. Decision: create a `buildAgentsForSimulation` top-level function in simulation_provider.dart that builds all-bot agents (no human player index 0 special case):
  ```dart
  Map<int, PlayerAgent> buildSimulationAgents(GameState state, MapGraph mapGraph, Difficulty difficulty) {
    return { for (int i = 0; i < state.players.length; i++) i: _makeBot(difficulty, mapGraph) };
  }
  PlayerAgent _makeBot(Difficulty d, MapGraph mg) => switch (d) {
    Difficulty.easy => EasyAgent(mapGraph: mg),
    Difficulty.medium => MediumAgent(mapGraph: mg),
    Difficulty.hard => HardAgent(mapGraph: mg),
  };
  ```
