### Human Verification Required

#### 1. App Lifecycle Save on Background (SAVE-01 on-hide trigger)

**Test:** Run `flutter run` on an Android emulator or physical device. Start a new game (tap "New Game (3 Players, Medium)"). Wait for the resume prompt showing "Turn 0". Press the home button to background the app. Relaunch from the app icon.
**Expected:** The app shows "Resume Game? Turn X" — confirming that `AppLifecycleListener.onHide` fired, `_saveState()` wrote to ObjectBox, and `build()` restored the state on cold start.
**Why human:** `AppLifecycleListener.onHide` is a Flutter engine lifecycle callback that fires when the OS moves the app to the background. It cannot be triggered in unit tests (requires a running Flutter engine with platform channel integration). WSL2 does not support Android emulator. The save/restore mechanism is fully verified by `saveNow()` unit tests against real ObjectBox; only the trigger wire (`onHide: _saveState`) needs live-device confirmation. The 09-03 SUMMARY notes the human checkpoint was approved by the user with the lifecycle test deferred to Phase 11 when a full game UI is available.
