<task type="auto" tdd="true">
  <name>Task 1: Combat resolution (single roll and blitz) with tests</name>
  <files>risk/engine/combat.py, tests/test_combat.py</files>
  <behavior>
    - resolve_combat(attacker_dice_count, defender_dice_count, rng) returns CombatResult with attacker_losses, defender_losses
    - Dice sorted descending before pairing: [6,3,1] vs [5,2] -> pairs (6,5) and (3,2)
    - Ties go to defender: (4,4) -> attacker loses
    - Attacker wins strict greater: (5,4) -> defender loses
    - validate_attack(state, map_graph, action) raises ValueError if: source not owned by current player, target not enemy, not adjacent, insufficient armies (need num_dice+1)
    - execute_attack(state, map_graph, action, rng) returns (new_state, CombatResult, conquered: bool)
    - On conquest: defender territory changes owner, attacker must move at least num_dice armies in, source retains at least 1
    - execute_blitz(state, map_graph, action, rng) loops combat until defender has 0 armies or attacker has 1 army
    - Blitz with seeded RNG produces deterministic results
    - After conquest, conquered_this_turn flag set to True on state
  </behavior>
  <action>
    Create risk/engine/combat.py:
    - CombatResult as Pydantic BaseModel: attacker_losses: int, defender_losses: int

    - resolve_combat(attacker_dice: int, defender_dice: int, rng: random.Random) -> CombatResult:
      Sort both roll lists descending, zip and compare, ties to defender. Return loss counts.

    - validate_attack(state: GameState, map_graph: MapGraph, action: AttackAction, player_index: int) -> None:
      Check source owned by player_index, target NOT owned by player_index, adjacency, source armies >= action.num_dice + 1. Raise ValueError with descriptive message on failure.

    - execute_attack(state: GameState, map_graph: MapGraph, action: AttackAction, player_index: int, rng: random.Random, defender_dice: int = None) -> tuple[GameState, CombatResult, bool]:
      Call validate_attack. Determine defender dice (default: min(2, defender_armies)). Call resolve_combat. Apply losses. If defender reaches 0 armies: transfer territory ownership, move num_dice armies from source to target, set conquered_this_turn=True. Return (new_state, result, conquered).

    - execute_blitz(state: GameState, map_graph: MapGraph, action: BlitzAction, player_index: int, rng: random.Random) -> tuple[GameState, list[CombatResult], bool]:
      Loop: determine max attacker dice (min(3, source_armies-1)), max defender dice (min(2, target_armies)), resolve combat, apply. Stop when target armies=0 (conquered) or source armies=1 (failed). Return (final_state, all_results, conquered).

    Create tests/test_combat.py:
    - test_dice_pairing_highest_first: seeded RNG producing known rolls, verify correct losses
    - test_ties_go_to_defender: mock/seed to get equal dice, verify attacker loses
    - test_attacker_wins_strict_greater: seed to get attacker higher, verify defender loses
    - test_validate_attack_wrong_owner: source owned by other player -> ValueError
    - test_validate_attack_not_adjacent: non-adjacent territories -> ValueError
    - test_validate_attack_insufficient_armies: 2 armies trying 2 dice -> ValueError (need 3)
    - test_validate_attack_target_friendly: target owned by same player -> ValueError
    - test_execute_attack_conquest: attacker wins, territory changes owner, armies moved
    - test_execute_attack_no_conquest: defender survives, armies reduced but ownership unchanged
    - test_execute_attack_min_army_remains: after conquest, source still has >= 1 army
    - test_blitz_attacker_wins: seeded so attacker overwhelms defender
    - test_blitz_attacker_fails: seeded so attacker reduced to 1 army
    - test_blitz_deterministic: same seed same result
    - test_conquered_flag_set: after conquest, state.conquered_this_turn is True
  </action>
  <verify>
    <automated>pytest tests/test_combat.py -x -v</automated>
  </verify>
  <done>Combat resolution correctly pairs dice highest-first, ties go to defender. Attack validation catches all invalid cases. Blitz loops correctly. Conquest transfers ownership and moves armies. All combat tests pass.</done>
</task>

  <name>Task 2: Fortification with connected-path validation and tests</name>
  <files>risk/engine/fortify.py, risk/engine/__init__.py, tests/test_fortify.py</files>
  <behavior>
    - validate_fortify(state, map_graph, action, player_index) raises ValueError if: source not owned by player, target not owned by player, target not reachable through connected friendly path from source, armies > source_armies - 1 (must leave 1)
    - execute_fortify(state, map_graph, action, player_index) returns new GameState with armies moved
    - Connected path works through chain of friendly territories (not just adjacent)
    - Rejects movement through enemy territory even if territories are eventually connected by another path
    - Moving all but 1 army is allowed (armies = source_armies - 1)
    - Moving 0 armies is rejected by FortifyAction validation (armies ge=1)
  </behavior>
  <action>
    Create risk/engine/fortify.py:
    - validate_fortify(state: GameState, map_graph: MapGraph, action: FortifyAction, player_index: int) -> None:
      Check source owned by player_index. Check target owned by player_index. Get player's territories set. Call map_graph.connected_territories(source, player_territories). Check target in connected set. Check action.armies <= source.armies - 1. Raise ValueError on failure.

    - execute_fortify(state: GameState, map_graph: MapGraph, action: FortifyAction, player_index: int) -> GameState:
      Call validate_fortify. Build new territories dict: source loses action.armies, target gains action.armies. Return state.model_copy(update={"territories": new_territories}).

    Update risk/engine/__init__.py: export resolve_combat, execute_attack, execute_blitz, validate_fortify, execute_fortify, and all card/reinforcement exports from Plan 01.

    - test_valid_fortify_adjacent: move armies between two adjacent friendly territories -> success
    - test_valid_fortify_through_chain: move armies through a chain of 3+ friendly territories -> success
    - test_fortify_blocked_by_enemy: two friendly territories connected only through enemy -> ValueError
    - test_fortify_source_not_owned: source owned by other player -> ValueError
    - test_fortify_target_not_owned: target owned by other player -> ValueError
    - test_fortify_too_many_armies: trying to move all armies (leaving 0) -> ValueError
    - test_fortify_max_armies: moving source_armies - 1 -> success, source has exactly 1
    - test_fortify_state_unchanged_on_error: after ValueError, original state unmodified
    - test_fortify_armies_correct: after fortify, source.armies = old - moved, target.armies = old + moved
  </action>
