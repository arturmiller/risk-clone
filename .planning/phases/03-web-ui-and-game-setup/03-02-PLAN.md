  <action>
    Create a schematic/simplified SVG world map where each of the 42 Risk territories is a `<path>` or `<polygon>` element. This is a functional game map, not a geographically accurate projection. The approach:

    1. Create `risk/data/classic_map.svg` as a ~1200x700 viewBox SVG.
    2. Each territory is a closed `<path>` element with:
       - `data-territory="ExactName"` attribute matching classic.json names exactly (e.g., `data-territory="Northwest Territory"`, `data-territory="Eastern Australia"`)
       - `class="territory"` for CSS styling
       - `id` attribute derived from territory name (e.g., id="territory-alaska")
       - Default fill of `#d4d4d4` (neutral gray)
       - Stroke of `#666` with 1px width
    3. Each territory also has a companion `<text>` element for the army count label:
       - `data-army-label="ExactName"` matching the territory name
       - Positioned at the approximate centroid of the territory path
       - Font size ~12px, font-weight bold, text-anchor middle
       - Default text content "0"
    4. Group territories by continent using `<g>` elements with `data-continent="ContinentName"`.
    5. Add continent labels as decorative text (smaller, italic) near each continent grouping.

    Territory layout (approximate world positions, schematic):
    - North America: top-left quadrant (Alaska top-left, Greenland top-right, Central America bottom)
    - South America: lower-left (Venezuela top, Argentina bottom)
    - Europe: top-center (Iceland top-left, Ukraine right)
    - Africa: center-left (North Africa top, South Africa/Madagascar bottom)
    - Asia: top-right spanning wide (Middle East left, Kamchatka far right, Siam bottom)
    - Australia: bottom-right (Indonesia left, Eastern Australia right)

    Use simple polygon shapes -- rectangles with rounded features, or basic outlined regions. Accuracy to real geography is NOT needed. Clarity and clickability ARE needed -- territories should be large enough to click and visually distinct.

    Verify all 42 territories are present by counting `data-territory` attributes. Verify names match classic.json exactly (case-sensitive).
